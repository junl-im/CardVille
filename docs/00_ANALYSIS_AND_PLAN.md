# CardVille v1 분석 및 개발 계획

## 1. 프로젝트 정체성

CardVille은 단순 단어 카드 게임이 아니라, 하나의 카드 엔진 위에서 여러 학습·퍼즐 모드를 계속 확장하는 수집형 카드 퍼즐 게임이다.

외부 서비스명은 `카드마을 CardVille`, 내부 엔진명은 `Dream Cards Engine`으로 구분한다.

## 2. 참고 이미지 분석

참고 게임의 강점은 다음과 같다.

- 모바일 세로 화면에 최적화되어 있다.
- 상단에 재화, 레벨, 설정이 즉시 보인다.
- 카드 스택과 단어 선택 구조가 직관적이다.
- 잠금 해제와 광고 보상 구조가 단순하다.
- 한 손 조작에 적합하다.

단점은 다음과 같다.

- 배경이 단색이라 프리미엄 감성이 약하다.
- 카드 프레임과 등급 차이가 약하다.
- 정답 연출이 약하면 보상감이 낮다.
- 단일 모드 구조라 장기 플레이 동기가 약하다.
- 카드 컬렉션, 앨범, 시즌 월드 같은 장기 목표가 부족하다.

## 3. CardVille 차별점

CardVille은 다음 차별점을 가진다.

- 낱말, 연산, 영어, 기억력, 색깔, 퍼즐, 초성, 그림, 음악 모드를 하나의 엔진에서 처리한다.
- 새로운 모드는 JSON 데이터 추가만으로 확장한다.
- 플레이 보상으로 카드팩을 획득한다.
- 카드팩에서 컬렉션 카드를 얻는다.
- 앨범 완성 보상으로 장기 플레이 목표를 만든다.
- 시즌별 월드로 배경과 카드 프레임을 교체한다.

## 7. 중요한 운영 원칙

무료 플랜 기준에서는 Firestore 읽기/쓰기를 아껴야 한다.

따라서 게임 모드 데이터는 Firestore가 아니라 정적 JSON 파일로 제공하고, Firestore는 유저 데이터 저장에 집중한다.

랭킹은 클라이언트 직접 쓰기 구조에서는 완전한 치팅 방지가 어렵다. v1에서는 참고용 랭킹만 사용하고, 경쟁 보상과 연결하지 않는다.
